# ファビコン選択と実装ガイド

勤怠管理キットアプリケーション用に、プラモデルのランナーフレームをモチーフにした10種類のSVGファビコンデザインを作成しました。

## 📂 ファイル配置

すべてのファビコンは以下の場所に配置されています：

```
apps/frontend/public/favicons/
├── favicon-01.svg ~ favicon-10.svg
├── preview.html
├── README.md
└── USAGE.md
```

## 🎨 デザインコンセプト

### プラモデルのランナーフレーム構造

各ファビコンは、プラモデルのランナーフレーム（組み立て前のパーツ配置枠）をモチーフにしています：

1. **外枠**: プラモデルのランナーフレームを模した四角い枠線
2. **内枠**: 追加の内側フレーム線
3. **バリ（ゲート）**: パーツをフレームから切り離す際の接続部分。各デザインで配置位置が異なります
4. **中央の大きな時計**: 勤怠管理の核心である時間要素
5. **中サイズのチェックマーク**: タスク完了や出退勤記録を表現
6. **小さな要素**:
   - **社員証（IDバッジ）**: 社員識別
   - **太陽**: 日中勤務を象徴
   - **月**: 夜勤や深夜勤務を象徴

### カラースキーム

すべてのファビコンは、アプリケーションのカラースキームを使用して作成されています：

- **プライマリカラー**: `#2563eb` (青) - 時計、社員証
- **成功カラー**: `#16a34a` (緑) - チェックマーク
- **セカンダリカラー**: `#64748b` (グレー) - フレーム、バリ
- **太陽**: `#fbbf24` (黄色)
- **月**: `#94a3b8` (薄いグレー)
- **ホワイト**: `#ffffff` - 時計の文字盤

## 🔍 プレビュー方法

### ローカル開発サーバーでプレビュー

開発サーバーを起動した状態で、以下のURLにアクセスしてください：

```
http://localhost:5173/favicons/preview.html
```

または、faviconディレクトリで直接HTTPサーバーを起動：

```bash
cd apps/frontend/public/favicons
python3 -m http.server 8080
# ブラウザで http://localhost:8080/preview.html を開く
```

## 📋 デザイン一覧

すべてのデザインは同じ基本構造（外枠、内枠、大きな時計、チェックマーク、社員証、太陽、月）を持ちますが、**バリ（ゲート）の配置位置**が異なります。これにより、それぞれ独自の個性を持ったデザインとなっています。

| ファイル名 | 配置パターン | 特徴 | 推奨用途 |
|-----------|-------------|------|---------|
| `favicon-01.svg` | 基本配置 | バリが上下左右中央に配置。最もバランスの取れた構成 | 汎用的、標準的な印象 |
| `favicon-02.svg` | 対角配置 | バリが対角線上に集中。非対称なレイアウト | ダイナミックな印象を与えたい場合 |
| `favicon-03.svg` | 変則配置 | バリが逆対角に配置。動的な印象 | 革新的、モダンな雰囲気 |
| `favicon-04.svg` | 複数バリ配置 | 上部に2つのバリ。安定感を重視 | 信頼性、安定性を強調したい場合 |
| `favicon-05.svg` | 非対称配置 | 左側にバリが多い。ユニークな構成 | 個性的なブランディング |
| `favicon-06.svg` | サイド強調配置 | 下部と右側に複数バリ。横長の印象 | 横方向の動きを意識する場合 |
| `favicon-07.svg` | バランス配置 | バリが複数箇所に分散。均等な印象 | 全体的な調和を重視する場合 |
| `favicon-08.svg` | 対角重視配置 | 上下に集中。縦長の印象 | 縦方向の流れを意識する場合 |
| `favicon-09.svg` | シンプル配置 | バリの数を最小限に。洗練された印象 | ミニマルデザインが好まれる場合 |
| `favicon-10.svg` | 四隅配置 | 四隅と左右中央にバリ。枠組み強調 | 堅牢性、構造を強調したい場合 |

## 🚀 実装方法

### ステップ1: ファビコンを選択

上記の一覧からお好みのデザインを選択してください。

### ステップ2: HTMLファイルを更新

`apps/frontend/index.html`の`<head>`セクションを更新します：

```html
<link rel="icon" type="image/svg+xml" href="/favicons/favicon-XX-name.svg" />
```

例（シンプル時計を選択した場合）：

```html
<link rel="icon" type="image/svg+xml" href="/favicons/favicon-01-clock-simple.svg" />
```

### ステップ3: 開発サーバーで確認

```bash
npm run dev -w @attendance-kit/frontend
```

ブラウザのタブでファビコンが正しく表示されることを確認してください。

## 💡 推奨デザイン

用途別の推奨デザイン：

### 🏢 企業向け汎用
- **favicon-01.svg** - 基本配置。バランスが良く、万人受けするデザイン
- **favicon-07.svg** - バランス配置。均等な印象で安定感がある

### 🎯 革新的・モダン
- **favicon-02.svg** - 対角配置。ダイナミックで現代的
- **favicon-03.svg** - 変則配置。革新的な印象

### 💼 堅実・信頼性重視
- **favicon-04.svg** - 複数バリ配置。安定感と信頼性
- **favicon-10.svg** - 四隅配置。構造の堅牢性を強調

### 🎨 個性的・ユニーク
- **favicon-05.svg** - 非対称配置。独自性の高いデザイン
- **favicon-09.svg** - シンプル配置。洗練されたミニマルな印象

### 📊 バランス重視
- **favicon-06.svg** - サイド強調配置。横方向のバランス
- **favicon-08.svg** - 対角重視配置。縦方向の流れ

## 🎯 選択のポイント

1. **ブランドイメージ**: 企業の印象に合った配置パターンを選択
2. **視認性**: 小さいサイズでも識別しやすいか
3. **個性**: 他社との差別化を図りたいか、標準的なデザインが良いか
4. **バリの配置**: 各デザインのバリの位置が持つ印象（対称性、動的、安定感など）を考慮

## 📝 カスタマイズ

SVGファイルはテキストエディタで直接編集可能です。色やサイズを調整したい場合は、該当のSVGファイルを開いて以下の値を変更してください：

- `fill="#2563eb"` - プライマリカラー
- `fill="#16a34a"` - 成功カラー
- `fill="#64748b"` - セカンダリカラー

## 🔄 複数サイズの生成

SVGは自動的にスケーリングされますが、PNGバージョンが必要な場合は以下のツールを使用できます：

- [SVGOMG](https://jakearchibald.github.io/svgomg/) - SVG最適化
- [CloudConvert](https://cloudconvert.com/svg-to-png) - SVG to PNG変換
