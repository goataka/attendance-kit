このページでは、勤怠管理システムの各機能について、画面のスクリーンショット付きで詳しく説明します。

## 初期画面

アプリケーションを開くと、以下の画面が表示されます。

![初期画面](/images/screenshots/01-initial-screen.png)

### 画面構成

- **ヘッダー**: 「勤怠管理システム」のタイトル（プライマリカラー: #007CC0）
- **入力エリア**: ユーザーIDと名前を入力するフィールド
- **操作ボタン**: 出勤打刻・退勤打刻ボタン
- **打刻履歴**: 過去の打刻記録を表示するテーブル

## 出勤打刻

### 手順

1. **ユーザーID入力**: 任意のユーザーIDを入力します（例: user001）
2. **名前入力**: 名前を入力します（例: 山田太郎）
3. **出勤打刻ボタンをクリック**: 青い「出勤打刻」ボタンをクリックします

### 実行結果

![出勤打刻後](/images/screenshots/02-after-clock-in.png)

打刻が完了すると、打刻履歴テーブルに新しいレコードが追加されます。

### 表示内容

打刻履歴テーブルには以下の情報が表示されます：

- **ユーザーID**: 打刻したユーザーのID
- **出勤時刻**: 出勤打刻を行った日時（YYYY/MM/DD HH:MM形式）
- **退勤時刻**: 退勤打刻を行った日時（未打刻の場合は「-」）
- **状態**: 
  - **勤務中** (青色バッジ): 出勤打刻済み、退勤打刻未実施
  - **完了** (緑色バッジ): 出勤・退勤の両方が完了

## 退勤打刻

### 手順

1. **ユーザーID入力**: 出勤時と同じユーザーIDを入力します
2. **退勤打刻ボタンをクリック**: 赤い「退勤打刻」ボタンをクリックします

### 実行結果

![退勤打刻後](/images/screenshots/03-after-clock-out.png)

退勤打刻が完了すると、該当レコードに退勤時刻が記録され、状態が「完了」に変わります。

### 表示の変化

- **退勤時刻**: 退勤打刻を行った日時が表示されます
- **状態**: 「勤務中」から「完了」に変わり、バッジの色が青から緑に変わります

## レスポンシブデザイン

このシステムは、さまざまな画面サイズに対応しています。

### 対応解像度

- **最低解像度**: 1280 x 1024 ピクセル
- **推奨解像度**: 1920 x 1080 ピクセル以上
- **モバイル対応**: スマートフォンでも利用可能（レスポンシブレイアウト）

### デバイス別レイアウト

#### デスクトップ (1920x1080以上)

- 横並びレイアウト
- 入力フィールドとボタンが横に配置
- 打刻履歴テーブルが広く表示

#### タブレット (1024x768程度)

- 2カラムレイアウト
- 入力フィールドが2列に配置
- ボタンは横並びを維持

#### スマートフォン (768px以下)

- 縦積みレイアウト
- 入力フィールドが1列に配置
- ボタンが縦に並ぶ
- テーブルがコンパクトに表示

## 注意事項

### データの永続化

現在のMVP版では、データはメモリ内に保存されます。以下の点に注意してください：

- **ページリロード**: ブラウザをリロードすると、すべての打刻データが消えます
- **サーバー再起動**: バックエンドサーバーを再起動すると、データが失われます

:::tip[今後の実装]
将来的には、DynamoDBなどのデータベースに打刻データを永続化する予定です。
:::

### 認証機能

現在のMVP版では、認証機能は実装されていません：

- **ユーザーID**: 自由に入力可能（検証なし）
- **名前**: 任意の名前を入力可能

:::caution[セキュリティ]
本番環境では、必ず認証機能を実装してください。Phase 2でAWS CognitoまたはOIDCによる認証を追加予定です。
:::

## トラブルシューティング

### 退勤打刻ができない

**症状**: 退勤打刻ボタンをクリックしても、「出勤打刻が見つかりません」というメッセージが表示される

**原因**: 
- 出勤打刻を行っていない
- 異なるユーザーIDで退勤打刻を試みている
- ページをリロードして、出勤打刻データが消えている

**解決方法**:
1. 同じユーザーIDで出勤打刻を行ってください
2. ページをリロードせずに、出勤打刻後すぐに退勤打刻を行ってください

### 打刻履歴が表示されない

**症状**: 打刻を行っても、履歴テーブルに何も表示されない

**原因**:
- ユーザーIDまたは名前が入力されていない
- バックエンドサーバーが起動していない

**解決方法**:
1. ユーザーIDと名前の両方が正しく入力されているか確認してください
2. バックエンドサーバーが起動しているか確認してください（`npm run dev:backend`）

## API仕様

このシステムは、以下のAPIを使用しています。

### 出勤打刻

```http
POST /api/clock-in
Content-Type: application/json

{
  "userId": "user001",
  "userName": "山田太郎"
}
```

**レスポンス**:
```json
{
  "success": true,
  "data": {
    "id": "1234567890",
    "userId": "user001",
    "clockInTime": "2026-01-04T11:08:00.000Z",
    "type": "clock-in"
  },
  "message": "Clock in successful"
}
```

### 退勤打刻

```http
POST /api/clock-out
Content-Type: application/json

{
  "userId": "user001"
}
```

**レスポンス**:
```json
{
  "success": true,
  "data": {
    "id": "1234567890",
    "userId": "user001",
    "clockInTime": "2026-01-04T11:08:00.000Z",
    "clockOutTime": "2026-01-04T11:09:00.000Z",
    "type": "clock-out"
  },
  "message": "Clock out successful"
}
```

### 打刻履歴取得

```http
GET /api/records?userId=user001
```

**レスポンス**:
```json
{
  "success": true,
  "data": [
    {
      "id": "1234567890",
      "userId": "user001",
      "clockInTime": "2026-01-04T11:08:00.000Z",
      "clockOutTime": "2026-01-04T11:09:00.000Z",
      "type": "clock-out"
    }
  ]
}
```
