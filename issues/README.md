# 課題管理

このフォルダはリポジトリ全体に共通する課題を管理します。

## プロジェクト別課題リンク

各プロジェクトの課題については、以下のリンクから参照してください：

- [Frontend](../apps/frontend/issues/)
- [Backend](../apps/backend/issues/)
- [Website](../apps/website/issues/)
- [Infrastructure](../infrastructure/issues/)

## 課題ファイルの作成方法

各課題は個別のmarkdownファイルとして作成してください。

### ファイル名の形式

`{課題名}.md` （例: `authentication.md`, `monitoring-setup.md`）

### ファイルの構成

各課題ファイルには以下のセクションを含めてください：

1. **要求仕様**: 課題の目的と要件
2. **実現案**: 技術的なアプローチとアーキテクチャ
3. **実装方式**: 具体的な実装手順
4. **タスク一覧**: 実装すべきタスクのチェックリスト
5. **進行状況**: 日付ごとの更新履歴
