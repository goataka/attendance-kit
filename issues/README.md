# 課題管理

このフォルダはリポジトリ全体に共通する課題を管理します。

## プロジェクト別課題リンク

各プロジェクトの課題については、以下のリンクから参照してください：

- [Frontend](../apps/frontend/issues/)
- [Backend](../apps/backend/issues/)
- [Website](../apps/website/issues/)
- [Infrastructure](../infrastructure/issues/)

## 共通課題

リポジトリ全体に関わる課題については、このフォルダ内にmarkdownファイルを作成して管理します。

### 課題管理の方法

1. 各プロジェクトのREADME.mdにタスク形式で課題を記載
2. 対応が必要な課題の詳細は、該当プロジェクトのissuesフォルダ内にmarkdownファイルを作成
3. markdownファイルには以下を記載：
   - 要求仕様
   - 実現案
   - 実装方式
   - タスクの一覧
   - 進行状況の更新
