# 課題ファイルの作成方法

**作成日**: 2026-01-13  
**ステータス**: 有効

## 1. ファイル名の形式

課題は個別のmarkdownファイルとして作成してください。

**命名規則**: `{課題名}.md`

**例**:
- `authentication.md` - 認証機能の実装
- `monitoring-setup.md` - 監視設定の構築

## 2. ファイルの構成

各課題ファイルには以下のセクションを含めてください：

### 2.1 要求仕様 *(mandatory)*

課題の目的と要件を記載します。

**含めるべき内容**:
- 機能要件: 実装すべき機能
- 非機能要件: パフォーマンス、セキュリティなど

### 2.2 実現案

技術的なアプローチとアーキテクチャを記載します。

**含めるべき内容**:
- アーキテクチャ図（Mermaid推奨）
- 技術スタック
- 設計判断の理由

### 2.3 実装方式

具体的な実装手順を記載します。

**含めるべき内容**:
- ファイル構成
- 実装手順（番号付きリスト）
- コードの配置場所

### 2.4 タスク一覧

実装すべきタスクをチェックリスト形式で記載します。

**形式**:
```markdown
- [ ] タスク名
  - サブタスク詳細
```

### 2.5 進行状況

日付ごとの更新履歴を記載します。

**形式**:
```markdown
### YYYY-MM-DD
- 完了した作業
- 決定事項

### 今後の予定
- 次のステップ
```
