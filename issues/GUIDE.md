# 課題ファイルの作成方法

**作成日**: 2026-01-13  
**更新日**: 2026-01-15  
**ステータス**: 有効

## 0. 課題の書き方

### 0.1 基本ルール

- **1行で記載**: 課題の本質を1行で簡潔に表現する
- **補足はインデント**: 追加情報は箇条書きでインデント
- **5行以内**: 1課題につき最大5行に収める
- **参考URLを記載**: 関連リソースがあれば追記
- **冗長さを排除**: 簡潔で明確な表現を心がける

### 0.2 記載例

```markdown
- [ ] markdown-lintをプレマージチェックに追加
  - 参考: [markdownlint-cli2](https://github.com/DavidAnson/markdownlint-cli2)

- [ ] shellcheckをプレマージチェックに追加
  - 参考: [ShellCheck](https://github.com/koalaman/shellcheck)
```

## 1. ファイル名の形式

- 命名規則: `YYYYMMDD-HH24MISS-{課題名}.md`
- 例: `20260113-143000-認証機能実装.md`
- 注: 個別ファイルは実際に着手する時のみ作成。それ以外はREADME.mdに箇条書きで記載し、詳細はインデントで追加。

## 2. ファイルの構成

### 2.1 タイトル

- 形式: `# {課題名} [ステータス]`
- 例: `# 認証機能実装 [実装中]`

**ユーザーシナリオ**:
- [シナリオ説明]
- **受け入れシナリオ**: **前提** [状態], **実行** [アクション], **結果** [期待結果]

**要件**:
- [機能] [要件]
- [非機能] [要件]

**成功基準**:
- [測定可能な成果]
